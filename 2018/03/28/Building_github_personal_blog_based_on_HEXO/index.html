Building_github_personal_blog_based_on_HEXO

Posted on 2018-03-28 | In Notes |
Words count in article: | Reading time ≈

导语

HEXO is a fast, simple and powerful blog framework. You write posts in Markdown (or other languages) and Hexo generates static fileswith a beautiful theme in seconds.

  作为一个正在成长的小coder，总喜欢搞一些事情，比如想着搭建一个自己的博客，来记录一些小笔记。用自己的服务器没搭出来，就借用了github的服务器，使用了现在大家使用频率较高的hexo+github来创建博客,而且hexo用户量大，主题还很多,对于比较注意表面的人，可以说很友好了。

  这篇文章是基于 Windows系统 对搭建过程做一个小的记录,虽然我的电脑已经提前将环境搭好了，但还是从零开始记录吧！

Ready~Go！


1.安装NodeJS

Node.js is an open-source, cross-platform JavaScript run-time environment that executes JavaScript code server-side

hexo是基于nodejs环境的静态博客，在接下来操作中，也会用到nodejs的npm安装。

点击进入NodeJS首页（说明：LTS为长期支持版，Current为当前最新版）

安装步骤：

首先下载好msi文件后，双击打开安装，也是一路next，不过在Custom Setup这一步记得选 Add to PATH ,这样你就不用自己去配置电脑上环境变量了，装完在按 win + r 快捷键调出运行，然后输入cmd，点击确定，在cmd中输入path可以看到你的node是否配置在里面（环境变量），没有的话你就自由发挥吧。

查看node版本

  在命令行输入：$node -v

2.安装Git

Git是一个开源的分布式版本控制系统，用以有效、高速的处理从很小到非常大的项目版本管理。

点击进入Git首页

  • 点击下载32-bit Git for Windows Setup
  • 点击下载64-bit Git for Windows Setup

安装步骤

双击下载好的.exe文件，一直点Next就好

查看Git版本

  打开Git Bash，输入命令：$git version

3.安装Hexo

Hexo is a fast, simple and powerful blog framework. You write posts in Markdown (or other languages) and Hexo generates static files with a beautiful theme in seconds.

 安装步骤

  •  先创建一个文件夹（用来存放所有blog的东西），然后cd到该文件夹下.
  •  安装hexo命令：$npm i -g hexo
  •  安装完成后，查看版本 :$hexo -v
  •  初始化命令：$hexo init

  •  初始化完成之 后打开所在的文件夹可以看到一些文件

     这里要解释一下:

    •   node_modules：是依赖包
      •   public：存放的是生成的页面
      •   scaffolds：命令生成文章等的模板
      •   source：用命令创建的各种文章
      •   themes：主题
      •   _config.yml：整个博客的配置
      •   db.json：source解析所得到的
      •   package.json：项目所需模块项目的配置信息

前置工作就此告一段落，接下来开始具体配置

4.链接GitHub

GitHub 是一个面向开源及私有 软件项目的托管平台，因为只支持 Git 作为唯一的版本库格式进行托管，故名 GitHub。

  • 首先，你得有一个GitHub的账号————如果你没有，赶紧创一个啊！

  • 然后，你要创建一个新的repo，名称为YourGitHubName.github.io必须按照这个格式创建才有用

  • 最后，回到Git Bash里面配置GitHub账户信息,如下（YourName和YourEmail都替换成你自己的）：

    $ git config –global user.name “YourName”
    $ git config –global user.email “YourEmail”

5.创建SSH并添加到GitHub中

（1）创建SSH

  • ·在Git Bash中输入：$ssh-keygen -t rsa -C “YourEmail@example.com“,生成ssh
  • ·在Git Bash中输入: $cd ~/.ssh
  • ·在Git Bash中输入：$cat id_ras.pub
  • ·你会在Git Bash中看到：
    ssh-ras这会有一段字符YourEmail@example.com这个样子的字符

（2）添加到GitHub

  • ·将上面获取到的SSH(ssh-ras和那一段字符，邮箱不要)到GitHub中————添加一个New SSH key ，title随便取，key就填刚刚那一段。
  • ·在Git Bash中验证是否添加成功:$ssh -T git@github.com

6.部署项目

  • 打开blog所在文件夹，修改_config.yml文件的一些配置,(冒号之后都是有一个半角空格的)：

    deploy:
     type: git
     repo:https://github.com/YourgithubName/YourgithubName.github.io.git 如果出现无法连接，可以修改为ssh://git@github.com/YourgithubName/YourgithubName.github.io
     branch: master

7.本地测试

  • 打开Git Bash 进入到blog目录，分别执行以下命令：

    • $hexo init
    • $hexo generate
    • $hexo server
    • _注：hexo 3.0把服务器独立成个别模块，需要单独安装：$npm i hexo-server。
  • 打开浏览器，在地址栏输入：http://localhost:4000，你就会看到Hexo默认界面了

8.上传到GitHub

  • 首先，安装一个小玩意，在Git Bash中输入：$npm install hexo-deployer-git –save(只有用它才能将文章部署到GitHub服务器，并且能被别人访问到)
  • 接下来在Git Bash输入命令(建议每次都按以下三步进行)：
    • $hexo clean
    • $hexo generate
    • $hexo deploy
    • 注：deploy过程中可能需要输入你的UserName和PassWord
  • 部署成功后，稍等一会，在自己浏览器输入http://YourGitHubName.github.io就可以看到你的个人博客啦!

基本操作已经完成，接下来都是附加操作~

9.绑定个人域名

有自己的博客，怎么能不配备一个自己喜欢的域名呢。

  • 首先，需要购买一个域名，比如我在阿里云购买的jarvissmile.top,DNS也是顺便在阿里云购买的，个人也比较支持阿里云！

  • 然后，在DNS解析中，增加一条记录

    记录类型为 A
    主机记录为www或者 @
    记录值为你GitHub的ip
    注：关于GitHub的IP地址，你可以打开CMD，输入：$ping YourGitHubName.github.io

  • 接下来，添加CNAME，在项目的source文件夹下面创建一个CNAME的文件，在里面添加自己的域名，比如jarvissmile.top注：只能添加一个

  • 再次部署，重复步骤8，然后在浏览器输入你的域名，就可以看到你的博客了~

10.修改及配置主题

 Hexo初始化之后默认的主题是landscape , 然后你可以去Hexo.io里面找到你想要的主题。在github中搜索你要的主题名称，里面都会有该主题的如何使用的介绍，按着来就好了，反正就是改改改！我选的是Claudia,看起来挺不错，至少是我喜欢的类型。
 跟主题相关的配置在themes/Claudia/_config.yml里面，每一个主题，应该都有Readme.md文档,可以参考，并对主题进行修改！

11.添加文章

  • 新建文章：$hexo new ‘文章名’，然后你就可以在source/_posts路径下看到你创建的文章啦，编辑完成之后按照前面说的方式部署，在浏览器刷新就能看到你的文章了。

  • 关于具体的文章编辑你可以看下官网介绍

  • 至于markdown，可以自行发挥啦~，推荐语法学习博客:Markdown——入门指南

附加操作也就到此结束
