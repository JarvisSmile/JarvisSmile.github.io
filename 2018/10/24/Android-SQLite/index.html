Let’s learn simple SQLite technology

SQLite介绍

  • SQLite是Android内置的一个数据库。

    Android提供了对 SQLite数据库的完全支持，应用程序中的任何类都可以通过名称来访问任何的数据库，但是应用程序之外的就不能访问。

  • SQLite学习地址: SQLite 教程 | 菜鸟教程

  • Android中，通过对SQLiteOpenHelper类对SQLite数据库进行操作。

SQLiteOpenHelper类介绍

  • 定义：SQLiteOpenHelper是一个辅助类

  • 功能：数据库基本操作（创建、增、删、改、查、更新） & 版本的控制

  • 用法：通过创建子类继承SQLiteOpenHelper类，实现它的一些方法来对数据库进行操作。

    在实际开发中，为了方便对数据库进行操作，我们会封装一个继承自SQLiteOpenHelper类的数据库操作类，然后以这个类为基础，再封装我们的逻辑方法。

  • 常用方法介绍(具体参数介绍在下一章节)

  • 数据库操作无外乎：“增删查改”：

    • 对于“增、删、改（更新）”这类对表内容变换的操作，需先调用getWritableDatabase()获得一个可写数据库对象，在执行时调用对应的操作API方法：insert()、delete()、update()，或者调用通用的execSQL(String sql) 变量sql是基本的sql语句
    • 对“查”，需要调用getReadableDatabase()获得一个可读的数据库对象，然后使用query()或rawQuery()方法

插入数据

  • 涉及方法：

    • getWritableDatabase()
    • insert(String table, String nullColumnHack, ContentValues values)
    • ContentValues.put(String key,String value);

  • insert(String table, String nullColumnHack, ContentValues values)

  • 变量分析

    • table：要操作的表名称
    • nullColumnHack：SQl不允许一个空列，如果ContentValues是空的，那么这一列被明确的指明为NULL值
    • values：ContentValues对象
  • ContentValues.put(String key,String value);
    • 变量分析
      • key：要插入的列名
      • value：要插入的值

        ContentValues内部实现就是HashMap，但是两者还是有差别的,ContenValues Key只能是String类型，Value只能存储基本类型数据，不能存储对象

删除数据

  • 涉及方法：

    • getWritableDatabase()
    • delete(String table, String whereClause, String[] whereArgs);

  • delete(String table, String whereClause, String[] whereArgs);

  • 变量分析

    • table：表名
    • whereClause：where选择语句, 选择哪些行要被删除, 如果为null, 就删除所有行;
    • whereArgs： where语句的参数, 逐个替换where语句中的 “?” 占位符;

      注，whereClause需要标明占位符，如：DB.delete(“user”,”id=?”,new String []{“1”});中 “id=?” ?就是占位符，方便whereArgs数据对应

修改数据

  • 涉及方法：

    • getWritableDatabase()
    • update(String table, ContentValues values, String whereClause, String[] whereArgs)

  • update(String table, ContentValues values, String whereClause, String[] whereArgs)

  • 变量分析

    • table：表名
    • values：ContentValues对象（需要修改的）
    • whereClause：where选择语句, 选择哪些行要被删除, 如果为null, 就修改所有行;
    • whereArgs： where语句的参数, 逐个替换where语句中的 “?” 占位符;

    注，whereClause需要标明占位符，如：DB.delete(“user”,”id=?”,new String []{“1”});中 “id=?” ?就是占位符，方便whereArgs数据对应

查询数据

  • 涉及方法

    • getReadableDatabase()
    • rawQuery(String sql, String[] selectionArgs)
    • query(String table, String[] columns, String selection, String[] selectionArgs, String groupBy, String having, String orderBy);
    • query(String table, String[] columns, String selection, String[] selectionArgs, String groupBy, String having, String orderBy, String limit);
    • query(String distinct, String table, String[] columns, String selection, String[] selectionArgs, String groupBy, String having, String orderBy, String limit);

  • rawQuery(String sql, String[] selectionArgs);

    • 变量分析
      • sql：将所有的SQL语句都组织到一个字符串中，使用占位符代替实际参数
      • selectionArgs：替换sql里面的占位符

  • query(String table, String[] columns, String selection, String[] selectionArgs, String groupBy, String having, String orderBy);

  • query(String table, String[] columns, String selection, String[] selectionArgs, String groupBy, String having, String orderBy, String limit);

  • query(String distinct, String table, String[] columns, String selection, String[] selectionArgs, String groupBy, String having, String orderBy, String limit);

  • 变量分析

    • table：要操作的表明
    • columns：查询的列所有名称集
    • selection：WHERE之后的条件语句，可以使用占位符
    • groupBy：指定分组的列名
    • having：指定分组条件，配合groupBy使用
    • orderBy：指定排序的列名
    • limit：指定分页参数

    • distinct：可以指定“true”或“false”表示要不要过滤重复值

      所有方法将返回一个Cursor对象，代表数据集的游标,数据分析时，需要调用Cursor的方法

      Cursor对象常用方法如下：

      Cursor.move(int offset);                 //以当前位置为参考,移动到指定行  
Cursor.moveToFirst();                    //移动到第一行  
Cursor.moveToLast();                     //移动到最后一行  
Cursor.moveToPosition(int position);     //移动到指定行  
Cursor.moveToPrevious();                 //移动到前一行  
Cursor.moveToNext();                     //移动到下一行  
Cursor.isFirst();                        //是否指向第一条  
Cursor.isLast();                         //是否指向最后一条  
Cursor.isBeforeFirst();                  //是否指向第一条之前  
Cursor.isAfterLast();                    //是否指向最后一条之后  
Cursor.isNull(int columnIndex);          //指定列是否为空(列基数为0)  
Cursor.isClosed();                       //游标是否已关闭  
Cursor.getCount();                       //总数据项数  
Cursor.getPosition();                    //返回当前游标所指向的行数  
Cursor.getColumnIndex(String columnName);//返回某列名对应的列索引值  
Cursor.getString(int columnIndex);       //返回当前行指定列的值

关于executeSQL()的使用

上述所有操作，除了查询操作(3.9)以外，都可以通过
db.executeSQL(String sql); 
db.executeSQL(String sql, Object[] bindArgs);
这两个函数代替，第二个函数中sql语句中使用占位符，然后第二个参数是实际的参数集

个人建议
尽量使用SQL语句,API方法参数较多，使用起来比较复杂。
